日志行配色体系

v0.7.0 设计图谱 · 同色系 · 透明度分层 · 图标识

主题

设计原则

  1. level 色是唯一语义色源。alert/error/warn/info/debug/trace 的 text 色（--c-*-text）是整个体系的"根色"。所有行状态——hover、marked（点击选中）、bookmarked——都从所属 level 的根色派生，而非另起固定色。这样一行 ERROR 无论在什么状态下都"看起来是红的"，语义不丢。
  2. 状态用透明度区分，不用色相。同一 level 色的不同透明度代表不同状态强度：hover ≈ 10%、marked ≈ 14%（加左侧条）、bookmarked ≈ 8%（加图标）。透明度越高 = 用户介入越深。这样色相统一、层次清晰。
  3. 图标承担状态识别，颜色不背这个锅。bookmarked 用 ★ 书签图标，marked 用左侧色条 + 选中图标。靠图标一眼区分"我收藏的"和"我点的"，而非靠两个相近的青蓝色去猜。
  4. 消除硬编码色。当前 is-copied(rgba(24,95,165))、is-highlighted(rgba(255,220,0))、is-marked(rgba(100,210,255)) 都是硬编码，不随主题变化。新体系统一用 color-mix(in srgb, var(--lv-*), var(--bg))，自动适配深浅主题。

1 色源：level 根色

整个体系从这里派生。根色 = 现有 --c-*-text，保持与 badge 文字、filterbar 标签一致。每行可直接交互：悬停看 hover 色，点击行切换 marked（选中），点击 ★ 切换 bookmarked。

同一 level 根色，不同透明度 = 不同状态。强度递增：默认(0%) → bookmarked(8%) → hover(10%) → marked(14%)。用户介入越深，底色越浓。

3 新旧对比

当前实现的核心问题：marked/bookmarked 用固定青蓝色，丢失 level 语义；多状态叠加时 hover 被覆盖。

现状（v0.6）问题

  • hover 按 level 区分色 --c-*-bg ✓ 有意义
  • 但 marked 用 rgba(100,210,255,0.3) 固定青蓝 ✗
  • bookmarked 用 rgba(100,210,255,0.15) 同色系青蓝 ✗
  • → ERROR 行被选中后变成青蓝色，"错误感"消失
  • → marked 和 bookmarked 都是青蓝，难以区分
  • → is-bookmarked 写在 CSS 最后，盖住 hover，书签行 hover 无反馈
  • → --c-marked-bg 根本没定义，永远走 fallback
  • → copied(蓝)/highlighted(黄) 硬编码，不随主题变

新方案（v0.7）解决

  • 所有状态都从 level 根色派生 color-mix(lv-*, %, bg)
  • → ERROR 行无论选中/书签，始终是红色系，语义保留
  • marked 14% + 左侧色条，bookmarked 8% + ★ 图标
  • → 靠透明度+图标区分，不靠两个相近色
  • hover 用复合选择器 .is-bookmarked:hover 叠加加深，反馈不丢
  • 统一 color-mix，深浅主题自动适配，零硬编码
  • 跳转高亮改用 level 根色脉冲，融入色系

4 CSS 映射（实现参考）

用 color-mix(in srgb, ...) 替代硬编码 rgba，无需为每个 level 重复声明。

选择器旧值新值
.log-row.level-error:hover var(--c-error-bg) color-mix(in srgb, var(--c-error-text) 10%, var(--bg))
.log-row.is-marked var(--c-marked-bg, rgba(100,210,255,.3))
固定青蓝		 删除此规则，改为按 level 动态着色：
.is-marked.level-error { background: color-mix(...,error 14%); }
.log-row.is-bookmarked var(--c-bookmark-bg, rgba(100,210,255,.3))
固定青蓝		 同上，按 level：color-mix(...,X 8%) + ★ 图标
.log-row.is-copied rgba(24,95,165,0.1) 硬编码 color-mix(in srgb, var(--accent) 10%, var(--bg))
.log-row.is-highlighted rgba(255,220,0,0.25) 硬编码黄 按 level：color-mix(...,X 14%) + 脉冲动画
叠加态 hover 被覆盖，无反馈 .is-bookmarked:hover { background: color-mix(...,X 16%); }

注：color-mix() 浏览器支持率 96%+（2024 起），Safari 16.2+/Chrome 111+/Firefox 113+。项目无 legacy 浏览器需求，可直接用。 也可用 CSS 变量预计算（--lv-cur）+ rgba 作为兜底。

5 搜索框 chip 色系

搜索关键词 chip 和日志行内 <mark> 高亮共用同一套色。 现状是 5 个硬编码 rgba 彩虹色（黄/青/品红/绿/橙），与 level 色系无关、不随主题变化。 新方案：用 HSL 色相分离保证多关键词可区分，透明度按主题调节保证可读。

新旧对比

现状（v0.6）问题

  • 5 个硬编码 rgba()：黄/青/品红/绿/橙
  • 颜色定义在 App.vue 的 highlightColors 数组里，非 CSS 变量
  • 深色主题下饱和度不足，高亮偏灰
  • 与 level 色系完全脱节
  • chip 和 <mark> 各用一份透明度，不一致

新方案（v0.7）解决

  • HSL 变量化：--kw-1 ~ --kw-5，色相值固定
  • 深浅主题各自调亮度/饱和度，暗底更鲜亮
  • chip 与 <mark> 共用同一 --kw-*，视觉统一
  • 透明度规则化（chip 14% / mark 30%），单一来源
  • 色相均匀分布在色环上（43°/200°/280°/150°/20°），多关键词易区分

6 应用框架配色

globalbar / statusbar / sidebar 三个区域的背景层次采用凹陷侧栏方案：sidebar 最深（--bg-2）、globalbar 极浅凹陷（text 3%）衬托 tab、主区域最亮（--bg）。鼠标移到侧边栏文件项看 hover，移到 tab 看 hover/active。
